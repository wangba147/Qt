QStyledItemDelegate 自定义委托

Qt Model/View 框架中的"自定义渲染引擎" —— 让列表项随心所欲地展示和编辑
Qt Widgets Model/View 2026-06-09

🧠 委托是什么？为什么需要它？

在 Qt 的 Model/View 架构中，View 负责展示，Model 负责数据，那"怎么展示"由谁决定？答案就是 Delegate（委托）。

💡 一句话总结：Delegate 是 View 和 Model 之间的"渲染插件"——Model 提供数据，Delegate 决定每一行、每一列如何在屏幕上画出来。

QStyledItemDelegate vs QItemDelegate

Qt 提供两个委托基类，QStyledItemDelegate 是 Qt 4.4 后推荐的默认选择：

特性QStyledItemDelegate 推荐QItemDelegate
样式集成✅ 完整支持 QSS 样式表❌ 不支持样式表
默认编辑器QDateTimeEdit / QDoubleSpinBox 等QLineEdit 为主
平台外观✅ 跟随系统主题经典风格
推荐场景现代 Qt 应用（99% 情况）旧版兼容 / 特殊需求

委托的核心职责（5个虚函数）

继承 QStyledItemDelegate 后，你可以重写以下虚函数来完全掌控渲染和编辑：

💠 渲染相关

  • paint() — 自定义绘制每一行/格
  • sizeHint() — 返回每行/格的推荐尺寸
  • displayText() Qt 6 — 自定义文本格式化

✏️ 编辑相关

  • createEditor() — 创建编辑控件
  • setEditorData() — 把 Model 数据装入编辑器
  • setModelData() — 把编辑结果写回 Model
  • updateEditorGeometry() — 调整编辑器位置/大小
💡 默认行为：如果不重写 paint()，QStyledItemDelegate 会用 QStyle 绘制标准的文本/图标/复选框——对大多数场景已经够用，只有需要"花里胡哨"时才自定义。

📝 三步接入委托

使用委托非常简单，只需创建、子类化、设置给 View：

步骤1：继承 QStyledItemDelegate

mydelegate.h #include <QStyledItemDelegate> class MyDelegate : public QStyledItemDelegate { Q_OBJECT public: explicit MyDelegate(QObject *parent = nullptr); // 核心重写 void paint(QPainter *painter, const QStyleOptionViewItem &option, const QModelIndex &index) const override; QSize sizeHint(const QStyleOptionViewItem &option, const QModelIndex &index) const override; };

步骤2：实现 paint() 和 sizeHint()

mydelegate.cpp void MyDelegate::paint(QPainter *painter, const QStyleOptionViewItem &option, const QModelIndex &index) const { // 1. 绘制选中/悬停背景（保持原生样式） QStyledItemDelegate::paint(painter, option, index); // 2. 获取数据 QString text = index.data(Qt::DisplayRole).toString(); // 3. 自定义绘制 painter->save(); painter->setPen(Qt::blue); painter->setFont(QFont("Arial", 12, QFont::Bold)); painter->drawText(option.rect.adjusted(5,0,0,0), Qt::AlignVCenter, text); painter->restore(); } QSize MyDelegate::sizeHint(const QStyleOptionViewItem &option, const QModelIndex &index) const { return QSize(200, 40); // 每行 200x40 像素 }

步骤3：设置给 View

mainwindow.cpp auto *listView = new QListView; auto *model = new QStringListModel; auto *delegate = new MyDelegate(this); listView->setModel(model); listView->setItemDelegate(delegate); // 🔑 关键一步 // 对于 QTableView，可分别为不同列设置不同委托 tableView->setItemDelegateForColumn(2, ratingDelegate); tableView->setItemDelegateForRow(0, headerDelegate);
💡 小技巧：QTableView 支持 按列、按行 甚至让某一列使用特殊的委托（如进度条列、星级评分列），非常灵活。

🎨 实战：多种自定义渲染效果

下面是一个完整委托类，演示了星级评分、进度条、优先级标签三种自定义渲染——在实际应用中非常常见。

完整委托类实现

ratingdelegate.h / .cpp class RatingDelegate : public QStyledItemDelegate { public: void paint(QPainter *p, const QStyleOptionViewItem &opt, const QModelIndex &idx) const override { // 先画背景（选中/悬停效果） QStyledItemDelegate::paint(p, opt, idx); p->save(); int col = idx.column(); if (col == 1) { // 星级评分列 ⭐ int stars = idx.data(Qt::DisplayRole).toInt(); QString starStr; for (int i = 0; i < 5; ++i) starStr += i < stars ? "★" : "☆"; p->setFont(QFont("Segoe UI Symbol", 16)); p->setPen(QColor("#f59e0b")); // 金色 p->drawText(opt.rect, Qt::AlignCenter, starStr); } else if (col == 2) { // 进度条列 📊 int pct = idx.data(Qt::DisplayRole).toInt(); QRect bar = opt.rect.adjusted(8,12,-8,-12); QRect fill = bar; fill.setWidth(bar.width() * pct / 100); // 背景 p->fillRect(bar, QColor(40,40,50)); // 填充 QColor fillColor = pct > 70 ? QColor("#10b981") : pct > 30 ? QColor("#0ea5e9") : QColor("#e94560"); p->fillRect(fill, fillColor); // 文字 p->setPen(Qt::white); p->drawText(bar, Qt::AlignCenter, QString("%1%").arg(pct)); } else if (col == 3) { // 优先级标签列 🏷️ QString prio = idx.data(Qt::DisplayRole).toString(); QColor bg, fg; if (prio == "高") { bg = QColor("#e94560"); fg = Qt::white; } else if (prio == "中") { bg = QColor("#f59e0b"); fg = QColor("#1a1a2e"); } else { bg = QColor("#10b981"); fg = Qt::white; } QRect tagRect = opt.rect.adjusted(16,10,-16,-10); p->setBrush(bg); p->setPen(Qt::NoPen); p->drawRoundedRect(tagRect, 6, 6); p->setPen(fg); p->drawText(tagRect, Qt::AlignCenter, prio); } p->restore(); } QSize sizeHint(const QStyleOptionViewItem &opt, const QModelIndex &idx) const override { return QSize(150, 44); // 更舒适的行高 } };

✏️ 自定义编辑器：让用户直接在表格里编辑

委托不仅能"画"，还能"编辑"。双击单元格时，委托会创建一个临时编辑器控件，编辑完成后把值写回 Model。

完整示例：星级评分双击编辑

stareditor.h / .cpp class StarDelegate : public QStyledItemDelegate { public: QWidget* createEditor(QWidget *parent, const QStyleOptionViewItem &opt, const QModelIndex &idx) const override { auto *combo = new QComboBox(parent); combo->addItems({"★☆☆☆☆","★★☆☆☆","★★★☆☆", "★★★★☆","★★★★★"}); combo->setCurrentIndex(2); // 默认3星 return combo; } void setEditorData(QWidget *editor, const QModelIndex &idx) const override { int stars = idx.data(Qt::EditRole).toInt(); auto *combo = qobject_cast<QComboBox*>(editor); if (combo) combo->setCurrentIndex(stars - 1); } void setModelData(QWidget *editor, QAbstractItemModel *model, const QModelIndex &idx) const override { auto *combo = qobject_cast<QComboBox*>(editor); if (combo) model->setData(idx, combo->currentIndex() + 1); } void updateEditorGeometry(QWidget *editor, const QStyleOptionViewItem &opt, const QModelIndex &idx) const override { editor->setGeometry(opt.rect); // 占满整个cell } };

🔴 关键信号：closeEditor()

编辑器提交时，View 会发出 closeEditor(QWidget*, QAbstractItemDelegate::EndEditHint) 信号。Delegate 也可以通过 commitData 和 closeEditor 信号主动控制提交时机：

// 在委托内部 emit 信号，通知 View 提交并关闭编辑器 emit commitData(editor); // 把编辑器值写入 Model emit closeEditor(editor, QAbstractItemDelegate::SubmitModelCache);

EndEditHint 枚举（决定编辑完后的焦点行为）：

  • NoHint — 无特殊行为
  • EditNextItem — 自动跳到下一行编辑
  • EditPreviousItem — 跳到上一行
  • SubmitModelCache — 提交缓存（默认）
  • RevertModelCache — 放弃缓存

⚠️ 7个常见陷阱与最佳实践

🚫 陷阱1：paint() 里忘记调用基类
如果不调用 QStyledItemDelegate::paint()，选中高亮、焦点框、悬停效果将全部消失，用户体验极差。务必在自定义绘制前先调基类。
⚠️ 陷阱2：createEditor 返回裸指针未设 parent
编辑器控件必须把 parent 参数传进去，否则编辑器由 View 管理生命周期时可能产生悬挂指针。正确做法：new QSpinBox(parent) 而非 new QSpinBox。
⚠️ 陷阱3：sizeHint 太小导致内容被裁切
如果自定义绘制了超大元素（图标、进度条、多行文本），务必在 sizeHint() 中返回足够尺寸，否则内容会被 View 裁剪。
🚫 陷阱4：在 setModelData() 中忘记检查 editor 类型
严格使用 qobject_cast 而不是 C 风格强转，防止在异常情况下崩溃。别忘了判空。
⚠️ 陷阱5：委托对象被提前析构
确保委托的生命周期覆盖 View（通常作为 View 的子对象，或使用成员变量）。View 不拥有委托的所有权，如果委托被提前 delete，View 会崩溃。
⚠️ 陷阱6：高频率更新导致性能问题
如果 Model 数据频繁变化（如实时数据），委托的 paint() 会被高频调用。应考虑：使用 QPainter 的裁剪跳过不可见区域、缓存复杂计算、降低不必要的重绘触发。
💡 陷阱7：paint() 中 save/restore 不匹配导致状态污染
始终用 painter->save() 和 painter->restore() 包裹自定义绘制代码，防止影响其他单元格的绘制状态。这是 QPainter 的黄金法则。

✅ 最佳实践清单

  1. 继承 QStyledItemDelegate 而非 QItemDelegate — 获得完整的 QSS 样式支持和平台原生外观
  2. paint() 开头调基类 → 再自定义绘制 — 保留选中/悬停/焦点等原生效果
  3. 用 QStyleOptionViewItem::rect 确定绘制区域 — 不要硬编码坐标，View 可能会 resize
  4. 重写 displayText() 而非在 paint() 里做文本格式化 — 语义更清晰，Qt 6 专属
  5. 为不同列使用不同委托 — 通过 setItemDelegateForColumn()，各司其职
  6. 用 openPersistentEditor 实现常驻编辑器 — 如复选框列、下拉框列始终可交互
  7. 复杂委托单元测试用 QTest — 直接调用 paint() 传入 QPixmap 验证像素级正确性
  8. 委托中不要缓存 Model 数据 — 一切从 index.data() 获取，Model 才是唯一真相来源
