Qt Graphics View 框架

QGraphicsScene · QGraphicsView · QGraphicsItem
2026-06-08 · Qt 6.8

概念与架构 — 三层模型

Graphics View 框架是 Qt 提供的高性能 2D 图形交互系统，基于 Model-View 思想设计，专门解决"大量可交互图形元素"的需求。

QGraphicsView 视口变换 · 交互事件 · 滚动缩放
↕ mapToScene / mapFromScene
QGraphicsScene BSP 树管理 · 碰撞检测 · 事件分发 · 渲染调度
↕ 管理 →
QGraphicsRectItem QGraphicsEllipseItem QGraphicsTextItem 自定义 Item ...

三层职责分离

层次类核心职责类比
视图层QGraphicsView可滚动视口、缩放/旋转/拖拽交互、坐标映射相机/取景器
场景层QGraphicsSceneItem 容器、BSP 树索引、碰撞检测、事件分发、渲染舞台/世界
元素层QGraphicsItem独立绘制、局部坐标、鼠标/键盘事件、父子变换演员/物体

为什么不用 QWidget 堆叠？

特性QWidget 堆叠Graphics View
最大 Item 数~ 几百个百万级
坐标变换手动 QPainterView 级内置矩阵
碰撞检测手动 O(n²)BSP 树 O(log n)
缩放/旋转手动计算view->scale() 一行
动画手动 QTimerQPropertyAnimation 直接驱动
核心思想： Scene 使用逻辑坐标（可任意定义），View 负责映射到屏幕像素。每个 Item 有独立局部坐标，父子间通过变换矩阵级联——这就是为什么它能轻松处理缩放旋转。

基础用法 — 三步上手

最小可运行示例

// 1. 创建场景（世界） — 逻辑坐标范围
QGraphicsScene scene(-200, -200, 400, 400);  // (x, y, w, h)

// 2. 添加图形项
auto *rect = scene.addRect(-50, -50, 100, 100,
    QPen(Qt::red, 2),
    QBrush(Qt::blue));
rect->setFlag(QGraphicsItem::ItemIsMovable);       // 可拖拽
rect->setFlag(QGraphicsItem::ItemIsSelectable);    // 可选中

// 3. 创建视图（窗口）
QGraphicsView view(&scene);
view.setRenderHint(QPainter::Antialiasing);
view.setDragMode(QGraphicsView::RubberBandDrag);   // 橡皮筋框选
view.show();

坐标体系

坐标系说明获取方式
Item 局部坐标相对于 Item 自身原点painter 在 paint() 内默认
Item 场景坐标相对于 Scene 原点item->scenePos()
View 坐标相对于 View 视口的像素鼠标事件的 event->pos()
// 坐标转换四件套
QPointF sp = item->mapToScene(localPt);     // 局部 → 场景
QPointF lp = item->mapFromScene(scenePt);   // 场景 → 局部
QPointF vp = view->mapFromScene(scenePt);   // 场景 → 视图
QPointF sp2 = view->mapToScene(viewPt);     // 视图 → 场景

视图变换 — 一行实现缩放旋转

view->scale(1.2, 1.2);                // 放大 20%
view->rotate(45);                      // 旋转 45°
view->centerOn(QPointF(100, 50));      // 滚动到指定位置
view->fitInView(scene.sceneRect(),        // 自适应显示全部
              Qt::KeepAspectRatio);
view->resetTransform();                 // 重置所有变换

自定义 Item — 必须重写的虚函数

虚函数说明必须?
boundingRect()返回 Item 的包围矩形（影响可见性判断、碰撞检测）必须
paint()绘制 Item 内容（唯一合法的绘制入口）必须
shape()定义精确命中区域（默认 = boundingRect）可选
type()返回自定义类型 ID，配合 qgraphicsitem_cast可选
itemChange()状态变化通知（位置、选中、可见性等）可选
class MyNode : public QGraphicsItem {
public:
    QRectF boundingRect() const override {
        return QRectF(-30, -30, 60, 60);    // 以原点为中心的 60x60 包围盒
    }

    void paint(QPainter *painter, const QStyleOptionGraphicsItem *,
               QWidget *) override {
        // 选中时变色
        painter->setBrush(isSelected() ? Qt::cyan : Qt::gray);
        painter->drawRoundedRect(boundingRect(), 10, 10);
        painter->drawText(boundingRect(), Qt::AlignCenter, m_label);
    }
private:
    QString m_label = "Node";
};

进阶技巧与避坑要点

进阶技巧

Item 分组 — QGraphicsItemGroup

auto *group = scene.createItemGroup({it1, it2, it3});
group->setFlags(QGraphicsItem::ItemIsMovable);
// 解散分组
scene.destroyItemGroup(group);

动画 — QPropertyAnimation

auto *anim = new QPropertyAnimation(item, "pos");
anim->setDuration(1000);
anim->setStartValue(QPointF(0, 0));
anim->setEndValue(QPointF(300, 200));
anim->start(QAbstractAnimation::DeleteWhenStopped);

嵌入 QWidget — QGraphicsProxyWidget

auto *proxy = scene.addWidget(
    new QPushButton("Click"));
proxy->setPos(100, 100);
proxy->setRotation(15); // 按钮也能旋转!
⚠ 每个 Widget 消耗系统资源，不宜大量使用

性能优化三板斧

// 1. 背景缓存
view->setCacheMode(QGraphicsView::CacheBackground);
// 2. Item 缓存
item->setCacheMode(QGraphicsItem::DeviceCoordinateCache);
// 3. 优化标志
view->setOptimizationFlags(
    QGraphicsView::DontSavePainterState
    | QGraphicsView::DontAdjustForAntialiasing);

8 个经典避坑要点

1
scene.clear() 会 delete 所有 Item！
外部持有的指针立即变野。
正确做法：用 removeItem() 逐个移除但保留 Item 所有权。
2
boundingRect() 包不住内容 → 不可见
必须包住所有绘制内容 + pen.widthF()/2 的笔宽溢出。
3
在 paint() 外调用 drawXXX 绘制
所有绘制 必须 在 paint() 虚函数内完成，否则不会显示。
4
Item 位置/大小改变前不调 prepareGeometryChange()
导致 BSP 树索引过时、碰撞检测错误、残留残影。位置/大小/变换改变前必须调用。
5
嵌入过多 QWidget 导致性能灾难
每个 QWidget 消耗系统窗口资源，应用崩溃时往往超过 ~50 个。
简单控件用自定义 Item 绘制替代。
6
忘记 setFlag(ItemIsFocusable) 导致键盘事件失效
需要键盘交互的 Item 必须设置此 Flag，否则按键事件不会路由到该 Item。
7
直接修改 pos() 而非 setPos() 触发不了 itemChange
始终使用 setPos() 移动 Item，它才会触发 itemChange(ItemPositionChange)。
8
多层缩放叠加导致浮点误差累积
长期使用 scale() 叠加会导致矩阵精度丢失。
定期用 resetTransform() + 重新计算目标变换矩阵。
性能警告： Graphics View 虽然支持百万级 Item，但大量 带动画的、频繁选中状态切换的、大号 QPixmap 的 Item 仍然会很慢。适时使用 DeviceCoordinateCache 缓存和限制可见 Item 数量。
